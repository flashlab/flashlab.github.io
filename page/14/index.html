wordpress-to-hexo

发表于 2015-04-22   |   分类于 原创

今天把之前wordpress备份的旧博文全部转成了markdown， 从09到13年，一共63篇。小站内容一下子丰富了不少～

首先简单处理了下图片见web-image-fix，上传到了七牛云。然后优雅地

什么？报错？！，看上去像是一段文内js迷惑了分析器。删掉重来，终于开始刷屏，最后还是抛出一个读不懂的错误orz。随便打开一个md文件，顿时傻眼，这是啥呀，简直就是一堆乱码！有木有！回想下，当时用wordpress装了一堆插件，居然还是用Windows Live Writer码的字。图一时爽快，搞得各种空格、转义符、span font标签、js碎片乱飞啊，格式完全乱套了，简直毫无头绪啊，有木有！好在文章不多，而且不用管评论，那就一篇篇改吧！（现在看见span就想吐）

这次下定决心，一定要把文章都弄成标准markdown，严格区分换行和分段，绝不放过任何多出的空格。然后放弃分类功能，只留标签。整理过程中还是有不少遗憾，许多代码和附件都找不回来，都怨那些短命的网盘！有些文章内容虽然已经过时，但还是尽量原汁原味保留，不为什么，情怀～～
现在终于懂了，关注内容才是博客生存的根本，你所做的，只是在一张白纸上铺开文字，仅此而已。

改造foo_uie_lyrics3，有心无力

发表于 2012-08-26   |   分类于 分享

用foobar2000有段时间了，对播放器唯一的要求就是要启动快，音质界面什么的都是次要的。所以只装了foo_uie_lyrics3一个插件用来看歌词。一直都想把 lyrics manager 整合进去，这样搜索的准确度和速度就大大提高了。今天无意中溜进了插件作者The vern 的发布帖，原来他早就想到了这点，还特地发布了一个自定义歌词来源的SDK（开源就是好）。赶紧下载下来，用VS2010试着编译了下，不出所料，报了一堆错误。一点点修改源码中的路径错误，终于顺利生成dll文件了。扔到components文件夹里，测试，有效。这说明加上lyrics manager 里的歌词源是完全可行的。正当暗自得意之时，突然发现自己对C++一窍不通。。。然后，然后就没下文了。。。

我把修改后的源码放上来，有兴趣的朋友可以试试。直接解压，用VS2010运行就可以了。
